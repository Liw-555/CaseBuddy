## Part 1: 什么是 AI Buddy / Claw？需要哪些组件？ {#part1}

### 1.1 概念定义

**AI Buddy**（也叫 AI Agent / AI Claw）= 一个能理解用户意图、调用工具完成复杂任务的 AI 助手。它不是一个简单的聊天机器人，而是一个**有工作流、有记忆、能执行多步骤任务**的智能体。

类比来说：
- **ChatBot** = 只会说话的嘴巴 🗣️
- **AI Buddy** = 有眼睛（看文件）、有手（调API）、有脑（分析推理）、有腿（连接多个平台）的完整助手 🤖

### 1.2 构建一个 AI Buddy 需要的 7 个核心组件

```

│                    AI Buddy 架构全景                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐         │
│  │  ① LLM   │  │ ② 前端UI │  │ ③ 后端API│  │ ④ 消息网关│         │
│  │ 大模型大脑 │  │ 用户交互  │  │ 业务逻辑  │  │ 多平台连接│       │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘         │
│       │            │             │             │                │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐         │
│  │ ⑤ RAG   │  │ ⑥ 工作流  │  │ ⑦ Bot   │  │            │          │
│  │ 知识检索  │  │ 任务编排  │  │ 多平台适配│  │          │        │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### ① LLM 大模型（大脑）
- **作用**：理解用户意图、生成分析内容、驱动决策
- **CaseBuddy 选型**：华东师大 ECNU 开放平台 `ecnu-plus` / `ecnu-max`
- **关键 API**：`POST /chat/completions`（兼容 OpenAI 格式）
- **技术约束**：
  - 不支持 `system` role，需合并到 user message
  - prompt 总长度超过 3000 字返回 500 错误
  - max_tokens 建议不超过 3000

#### ② 前端 UI（用户交互层）
- **作用**：提供用户友好的交互界面，包括文件上传、对话、结果展示
- **技术栈**：React 18 + TypeScript + Tailwind CSS + Vite
- **核心设计**：
  - 左侧导航栏（可折叠）+ 右侧内容区
  - 会话管理系统（SessionContext）
  - 实时流式输出（SSE）

#### ③ 后端 API（业务逻辑层）
- **作用**：处理业务逻辑、调用 LLM、管理数据、代理网关请求
- **技术栈**：Node.js + Express + TypeScript
- **核心能力**：
  - LLM 代理（非流式 + SSE 流式）
  - 文件解析（PDF / DOCX）
  - PPT 生成（pptxgenjs）
  - RAG 知识库（BM25 检索）
  - 工作流引擎

#### ④ 消息网关（多平台连接层）
- **作用**：统一管理微信、飞书、企业微信、QQ 等平台的连接
- **技术栈**：Python Flask + 各平台 SDK
- **核心设计**：HTTP API 服务，后端通过 `proxyToPython()` 代理请求

#### ⑤ RAG 知识检索（知识增强）
- **作用**：将长文档分块索引，按需检索相关片段注入 prompt
- **技术方案**：Node.js 内置 BM25 关键词检索（无 Python 依赖）
- **流程**：文档 → 分块(1000字/块,100字重叠) → 索引存储 → 关键词检索 → 上下文注入

#### ⑥ 工作流引擎（任务编排）
- **作用**：将复杂任务拆解为多步骤自动执行
- **预设模板**：案例速读、SWOT 分析、深度洞察、PPT 大纲、全流程
- **核心设计**：步骤状态机 (pending → running → completed/failed) + 上下文传递

#### ⑦ Bot 适配器（多平台 Bot）
- **作用**：让 AI 能在微信、飞书、企业微信等平台上收发消息
- **已实现**：
  - `wechat_bot.py` — 微信（基于 iLinkai 协议）
  - `feishu_bot_enhanced.py` — 飞书（基于 lark_oapi SDK）
  - `wecom_bot.py` — 企业微信
  - `qq_bot.py` — QQ

### 1.3 组件之间的协作关系

```
用户 ──→ 前端 ──→ 后端 API ──→ LLM
                      │
                      ├──→ Python 网关 ──→ 微信/飞书/QQ Bot
                      │
                      ├──→ RAG 知识库 ──→ 上下文增强
                      │
                      └──→ 工作流引擎 ──→ 多步骤编排
```

### 3.2 分析工作台 (WorkBench) {#mod-workbench}

**功能**：核心对话模块，支持 PDF 上传解析、多轮对话、技能调用、结果导出。

#### 核心功能

| 功能 | 实现方式 |
|------|----------|
| PDF/DOCX 上传 | `multer` + `pdf-parse` + `mammoth`，后端 `/api/parse/file` |

| 推送微信 | `POST /api/gateway/push-wechat` |

```typescript
  fileFilter: (_req, file, cb) => {
    const allowed = ['application/pdf', 'application/vnd.openxmlformats-officedocument.wordprocessingml.document'];

// 文件解析端点
app.post('/api/parse/file', upload.single('file'), async (req, res) => {
  if (mimetype === 'application/pdf') {
    const parser = new PDFParse({ data: new Uint8Array(buffer) });
    const result = await parser.getText();
    text = result.text;
  } else {
    const result = await mammoth.extractRawText({ buffer });
    text = result.value;
  }
  // 截断到 30000 字符，防止 prompt 过长

    text = text.slice(0, 30000) + '\n\n[... 文本已截断 ...]';
  }
```

#### SSE 流式输出 — 前端关键代码

> **文件路径**：`WorkBench.tsx`

```tsx
// 流式调用 LLM，逐字显示
const response = await fetch(`${API_BASE}/proxy/chat/completions/stream`, {
  method: 'POST',
  body: JSON.stringify({ baseUrl, apiKey, model, messages, temperature, max_tokens }),
});

const reader = response.body.getReader();
const decoder = new TextDecoder();
let buffer = '';

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  buffer += decoder.decode(value, { stream: true });
  // 解析 SSE data: 前缀
  const lines = buffer.split('\n');
  buffer = lines.pop() || '';
  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const data = line.slice(6);
      if (data === '[DONE]') break;
      // 解析 JSON，提取 content，追加到消息
    }
  }
}
```

#### 后端 SSE 代理 — 关键代码

> **文件路径**：`backend/src/index.ts`（第 79-148 行）

```typescript
app.post('/api/proxy/chat/completions/stream', async (req, res) => {
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');

  for await (const chunk of response.body as AsyncIterable<Uint8Array>) {
    buffer += decoder.decode(chunk, { stream: true });
    const lines = buffer.split('\n');
    buffer = lines.pop() || '';
    for (const line of lines) {
      if (line.startsWith('data: ')) {
        const data = line.slice(6);
        if (data === '[DONE]') { res.write('data: [DONE]\n\n'); res.end(); return; }
        res.write(`data: ${data}\n\n`);
      }
    }
  }
});
```

#### PPT 大纲生成流程

```
用户输入(分析内容 + 风格 + 页数)
    ↓
构建 Prompt（ECNU 兼容格式，控制长度）
    ↓
调用 LLM → JSON 格式的 slides 数组
    ↓
多层 JSON 解析（code block → direct JSON → 正则提取 → 截断修复）
    ↓
```

> **文件路径**：`backend/src/routes/pptCompare.ts`

```typescript
import JSZip from 'jszip';

async function parsePptx(filePath: string): Promise<string> {
  const fileBuffer = fs.readFileSync(filePath);
  const zip = await JSZip.loadAsync(fileBuffer);

  // 提取所有幻灯片文本
  const slideFiles: { path: string; num: number }[] = [];
  zip.forEach((relativePath) => {
    const match = relativePath.match(/^ppt\/slides\/slide(\d+)\.xml$/);
    if (match) slideFiles.push({ path: relativePath, num: parseInt(match[1]) });
  });
  slideFiles.sort((a, b) => a.num - b.num);  // 按编号排序

  // 正则提取 <a:t> 文本节点
  const textRegex = /<a:t[^>]*>([\s\S]*?)<\/a:t>/g;
  for (const sf of slideFiles) {
    const xml = await zip.file(sf.path)?.async('string');
    let match; while ((match = textRegex.exec(xml)) !== null) {
      texts.push(match[1]);
    }
  }
  return texts.join('\n');
}
```

**技术亮点**：`.pptx` 文件本质是 ZIP 包，里面是 XML 文件。直接用 JSZip 解压 + 正则提取文本，无需安装 `python-pptx`。

```typescript
async function executeWorkflow(workflow: Workflow) {
  const context: Record<string, any> = { parsedText: workflow.parsedText };

  for (const step of workflow.steps) {
    step.status = 'running';
    switch (step.type) {
      case 'parse_file':    // 使用已解析的文本
      case 'rag_index':     // 调用 chunkTextNode + keywordSearch
      case 'llm_analyze':   // 调用 LLM API（可选 RAG 增强）
      case 'ppt_outline':   // LLM 生成 PPT 大纲 JSON
      case 'ppt_generate':  // pptxgenjs 生成真实 .pptx 文件
    }
    // 每步完成后立即写入 results，前端轮询可实时看到进度
    workflow.results[step.name] = context[step.name];
  }
  workflow.status = 'completed';
}
```

```typescript
const PYTHON_GATEWAY_URL = 'http://127.0.0.1:3002';

// 通用代理函数 — 所有网关请求都经过这里
async function proxyToPython(endpoint, method = 'GET', body?) {
  const resp = await fetch(`${PYTHON_GATEWAY_URL}/${endpoint}`, {

    headers: { 'Content-Type': 'application/json' },
```
