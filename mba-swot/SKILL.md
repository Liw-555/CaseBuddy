# MBA案例分析 — SWOT + TOWS战略推导

## 使用场景

当用户提供企业案例材料（PDF/文字），要求进行 SWOT 分析、内外部环境分析、战略机会分析时使用此框架。

---

## 分析流程（严格按此顺序）

### Step 1：信息提取与归类

阅读案例材料，将信息按以下四维度归类：

| 维度 | 来源 | 核心问题 |
|------|------|----------|
| **S 优势（Strengths）** | 内部 | 企业相对竞争对手有哪些独特能力/资源？ |
| **W 劣势（Weaknesses）** | 内部 | 企业在哪些方面明显不足或受限？ |
| **O 机会（Opportunities）** | 外部 | 行业/市场/政策/技术趋势中有哪些有利时机？ |
| **T 威胁（Threats）** | 外部 | 竞争/监管/替代品/宏观环境中有哪些不利因素？ |

**每个维度至少列举3-5条，每条须有案例数据或事实支撑，禁止泛泛而谈。**

---

### Step 2：输出SWOT四象限

使用以下格式输出（Markdown表格）：

```
## SWOT 分析矩阵

| 内部优势 (S) | 内部劣势 (W) |
|---|---|
| S1: [具体优势+数据支撑] | W1: [具体劣势+影响] |
| S2: ... | W2: ... |
| S3: ... | W3: ... |

| 外部机会 (O) | 外部威胁 (T) |
|---|---|
| O1: [机会+来源] | T1: [威胁+严重程度] |
| O2: ... | T2: ... |
| O3: ... | T3: ... |
```

---

### Step 3：TOWS战略推导矩阵

基于SWOT推导四类战略（这是SWOT进阶——麦肯锡/哈佛 MBA 常考）：

| | 机会 (O) | 威胁 (T) |
|---|---|---|
| **优势 (S)** | **SO战略（进攻）**：用优势抓住机会 | **ST战略（防御）**：用优势抵御威胁 |
| **劣势 (W)** | **WO战略（转型）**：借机会弥补劣势 | **WT战略（退缩）**：规避威胁+掩盖劣势 |

每类战略至少给出2条具体策略建议，须与企业实际情况挂钩。

---

### Step 4：战略优先级建议

给出一个综合判断：
1. 当前最关键的战略方向（SO/ST/WO/WT中哪类优先？）
2. 理由（3句话以内，结论先行）
3. 最需要警惕的1个威胁 + 最值得把握的1个机会

---

## 输出格式模板

```markdown
# [公司名] SWOT分析报告

## 一、SWOT矩阵

[Step 2的表格]

## 二、TOWS战略推导

### SO战略（进攻型）
- 策略1：...
- 策略2：...

### ST战略（防御型）
- 策略1：...
- 策略2：...

### WO战略（转型型）
- 策略1：...
- 策略2：...

### WT战略（规避型）
- 策略1：...
- 策略2：...

## 三、战略优先级建议

**核心判断**：[结论一句话]

**理由**：
1. ...
2. ...

**最大威胁**：[T中第X条] — 原因：...
**核心机会**：[O中第X条] — 把握路径：...
```

---

## 质量标准

- 每条SWOT要素必须有**案例原文依据**，不得凭空捏造
- 优势/劣势必须是**相对于竞争对手**的，不是绝对描述
- 战略建议必须**可执行**，不能是空话（如"加强创新"须具体化为"在X领域通过Y方式实现Z目标"）
- 全文用**结论先行**的麦肯锡风格
