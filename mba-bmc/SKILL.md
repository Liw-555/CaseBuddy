# MBA案例分析 — 商业模式画布（BMC）

> 来源：亚历山大·奥斯特瓦德《商业模式新生代》（2010）。BMC用于**可视化和评估企业完整的价值创造逻辑**，是理解企业"如何赚钱"的最佳工具。

---

## BMC九宫格结构

```
┌─────────────┬────────────────────┬─────────────┬────────────────┐
│             │                    │             │                │
│  关键合作   │    关键活动         │  价值主张   │  客户关系      │
│ (KP)        │    (KA)            │  (VP)       │  (CR)          │
│             ├────────────────────┤             ├────────────────┤
│             │    核心资源         │             │  渠道通路      │
│             │    (KR)            │             │  (CH)          │
├─────────────┴────────────────────┴─────────────┴────────────────┤
│  成本结构 (CS)                    │  收入来源 (RS)               │
└──────────────────────────────────┴──────────────────────────────┘
                                    客户细分 (CS) 在右侧
```

九要素完整版：
1. **客户细分 (Customer Segments, CS)** — 为谁创造价值？
2. **价值主张 (Value Propositions, VP)** — 提供什么价值？解决什么问题？
3. **渠道通路 (Channels, CH)** — 如何传递价值？
4. **客户关系 (Customer Relationships, CR)** — 如何建立和维持客户关系？
5. **收入来源 (Revenue Streams, RS)** — 如何赚钱？
6. **核心资源 (Key Resources, KR)** — 需要哪些关键资产？
7. **关键活动 (Key Activities, KA)** — 必须做哪些最重要的事？
8. **关键合作 (Key Partnerships, KP)** — 哪些可以外包？依赖哪些合作伙伴？
9. **成本结构 (Cost Structure, CS)** — 最重要的成本是什么？

---

## 分析流程

### Step 1：逐模块填写

按以下顺序分析（**VP是核心，先分析VP再向两侧展开**）：

#### 优先分析顺序
```
VP（价值主张）→ CS（客户细分）→ RS（收入来源）
→ KR（核心资源）→ KA（关键活动）→ CH（渠道通路）
→ CR（客户关系）→ KP（关键合作）→ CS（成本结构）
```

#### 每个模块的分析要点

**1. 价值主张 (VP)**
- 为客户解决了什么痛点？（Pain Reliever）
- 为客户创造了什么增益？（Gain Creator）
- 核心差异化点是什么？（相比替代品）
- 用一句话表述：[为X客户，通过Y方式，提供Z价值]

**2. 客户细分 (CS)**
- 主要客户群体（可能有多个）
- 每个客户群的特征、规模、需求
- 大众市场 / 利基市场 / 分层市场 / 多边平台？

**3. 收入来源 (RS)**
- 收费模式：一次性买断/订阅/使用费/广告/佣金/授权？
- 定价逻辑：固定/动态/拍卖？
- 各收入来源占比（如有数据）

**4. 核心资源 (KR)**
- 实物资源（工厂/设备）
- 知识资源（专利/品牌/数据）
- 人力资源（关键人才）
- 财务资源（资金/信用额度）

**5. 关键活动 (KA)**
- 生产型 / 解决方案型 / 平台/网络型 中的哪类？
- 列出2-3项最核心的活动

**6. 渠道通路 (CH)**
- 获客渠道（线上/线下/合作）
- 交付渠道（如何让客户用到产品）
- 自有渠道 vs 合作渠道

**7. 客户关系 (CR)**
- 自助服务 / 自动化服务 / 专属服务 / 社区？
- 获取新客 vs 留存老客 vs 促进购买？

**8. 关键合作 (KP)**
- 战略联盟（非竞争者合作）
- 竞合关系（与竞争者合作）
- 外包活动（降低成本/风险）
- 关键供应商

**9. 成本结构 (CS)**
- 成本驱动型 vs 价值驱动型？
- 固定成本 vs 可变成本占比
- 最大的成本中心是什么？
- 规模经济/范围经济是否存在？

---

### Step 2：一致性检验

BMC九要素必须**相互支撑**，检查以下逻辑链：

```
价值主张 → 能否被核心资源和关键活动支撑？
渠道通路 → 是否匹配客户细分的获取习惯？
收入来源 → 是否足以覆盖成本结构？
关键合作 → 是否弥补了核心资源/活动的缺口？
```

如发现不一致，在报告中明确指出为**商业模式风险点**。

---

### Step 3：商业模式类型判断

判断属于以下哪类原型（奥斯特瓦德的商业模式类型）：

| 类型 | 特征 | 典型案例 |
|------|------|----------|
| **长尾模式** | 大量利基产品/低成本服务海量用户 | Netflix、亚马逊 |
| **多边平台** | 连接两个以上用户群体、网络效应 | 淘宝、Airbnb |
| **免费增值** | 基础免费 + 高级付费 | Dropbox、Spotify |
| **开放式模式** | 外部创新内化 / 内部资产外部变现 | 宝洁、ARM |
| **资产出售** | 出售产品所有权 | 传统制造业 |
| **订阅模式** | 持续收费获取持续服务 | SaaS、会员制 |

---

### Step 4：创新机会识别

基于BMC，指出：
1. **薄弱模块**：哪个模块当前最脆弱，存在风险？
2. **创新空间**：哪个模块有改进/颠覆的机会？
3. **竞争对比**：若有竞争对手数据，对比关键差异在哪里？

---

## 注意事项

- VP（价值主张）是整个画布的**核心**，必须最深入分析
- BMC是**描述性工具**，本身不判断好坏，好坏判断在Step 2一致性检验中
- 平台型企业（多边市场）有**多个客户细分**，需要为每个客群单独分析
- 收入来源与成本结构的比较要做出**盈利逻辑**的判断
